# 🏁 F1 대시보드 프로젝트 개발일지 - 첫 번째 게시물

안녕하세요!

오늘은 최근에 완성한 **F1 대시보드 웹 애플리케이션**을 소개하고 싶습니다.

## 📋 프로젝트 개요

포뮬러1 레이스 일정, 최근 결과, 챔피언십 순위, 드라이버 정보를 한눈에 볼 수 있는 대시보드입니다. Next.js와 TypeScript로 개발했으며, OpenF1 API를 활용해 실시간 데이터를 제공합니다.

## 🛠️ 기술 스택 선택 이유

### Next.js 14 (App Router)

- **Server Components**를 활용한 효율적인 데이터 페칭
- **자동 코드 스플리팅**으로 최적화된 번들 크기
- **SEO 친화적** 구조로 향후 확장성 고려

### TypeScript

- 타입 안정성으로 런타임 에러 사전 방지
- API 응답 데이터의 인터페이스 정의로 개발 생산성 향상
- 팀 협업 시 코드 가독성 및 유지보수성 개선

### Tailwind CSS

- 빠른 프로토타이핑과 일관된 디자인 시스템 구축
- 다크 모드 구현이 간단하고 직관적
- 반응형 디자인을 유틸리티 클래스로 쉽게 처리

## ✨ 주요 기능 및 구현 과정

### 1. 실시간 포디움 시각화

최근 레이스의 포디움을 3D 효과와 애니메이션으로 구현했습니다. 각 드라이버의 팀 컬러를 활용한 시각적 표현과 호버 효과를 추가해 사용자 경험을 개선했습니다.

```typescript
// 포디움 높이를 위치에 따라 동적으로 설정
const getPositionHeight = (position: number) => {
  switch (position) {
    case 1:
      return "h-40"; // 가장 높음
    case 2:
      return "h-32"; // 중간
    case 3:
      return "h-28"; // 가장 낮음
  }
};
```

### 2. OpenF1 API 데이터 변환 및 한글화

OpenF1 API의 영문 데이터를 한글로 변환하는 매핑 시스템을 구축했습니다. 드라이버 이름, 팀 이름, 국가 코드를 모두 한글로 번역하여 국내 사용자에게 친숙한 경험을 제공합니다.

**주요 구현 포인트:**

- 드라이버 이름, 팀 이름, 국가 코드 매핑 객체 생성
- API 응답 데이터를 앱 형식으로 변환하는 `transformOpenF1Driver` 함수
- 재사용 가능한 유틸리티 함수들 (`translateDriverName`, `getDriverHeadshot`)

### 3. 챔피언십 순위 시각화

드라이버와 컨스트럭터 순위를 탭으로 구분하여 표시하고, 각 드라이버의 포인트를 시각적 프로그레스 바로 표현했습니다. 팀 컬러를 활용한 디자인으로 브랜드 아이덴티티도 반영했습니다.

### 4. 반응형 디자인

모바일부터 데스크톱까지 모든 화면 크기에 최적화된 레이아웃을 구현했습니다. Tailwind의 그리드 시스템과 반응형 유틸리티를 활용해 유연한 구조를 만들었습니다.

## 🎨 디자인 철학

### 다크 모드 중심 디자인

F1의 고속감과 미래지향적인 느낌을 살리기 위해 다크 모드를 기본으로 설정했습니다. 그라데이션과 글로우 효과를 활용해 역동적인 분위기를 연출했습니다.

### F1 공식 컬러 팔레트

각 팀의 공식 컬러를 활용해 브랜드 아이덴티티를 반영했습니다. 레드불의 파란색, 페라리의 빨간색, 맥라렌의 오렌지 등 팀별 고유 컬러를 시각적 요소에 적용했습니다.

## 🚀 배운 점 & 어려웠던 점

### 배운 점

1. **Next.js Server Components의 강력함**: 클라이언트 사이드 렌더링 없이도 데이터를 효율적으로 가져올 수 있어 성능이 크게 개선되었습니다.
2. **TypeScript의 타입 가드**: API 응답 데이터의 타입을 명확히 정의함으로써 개발 중 발생할 수 있는 오류를 사전에 방지할 수 있었습니다.
3. **Tailwind CSS의 유틸리티 우선 접근**: 빠른 스타일링과 유지보수가 쉬워 개발 속도가 향상되었습니다.

### 어려웠던 점

1. **OpenF1 API 데이터 구조 파악**: 공식 문서가 제한적이어서 실제 API 응답을 분석하며 데이터 구조를 이해하는 데 시간이 걸렸습니다.
2. **한글 번역 매핑 관리**: 드라이버와 팀 이름이 매 시즌마다 변경될 수 있어 확장 가능한 구조로 설계하는 것이 중요했습니다.
3. **애니메이션 성능 최적화**: 여러 애니메이션을 동시에 실행할 때 성능 이슈가 있었지만, CSS 애니메이션과 `will-change` 속성을 활용해 해결했습니다.

## 📈 앞으로의 계획

1. **실시간 레이스 데이터 연동**: WebSocket을 활용한 실시간 랩 타임, 순위 업데이트
2. **드라이버 상세 페이지**: 각 드라이버의 통계, 경력, 최근 성적을 상세히 보여주는 페이지
3. **레이스 예측 기능**: 머신러닝을 활용한 레이스 결과 예측 시스템
4. **알림 기능**: 관심 있는 드라이버나 레이스의 알림 설정
